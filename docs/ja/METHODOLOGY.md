# 方法論

## 1. 観測単位

公開データの各行は1フェーズです。同一ポゼッションは、試合内の時系列、保持チーム、`Phase`の連続性、得点結果を使って再構成します。

ポゼッションの途中フェーズには最終結果を直接付与しません。最後のフェーズだけを吸収結果へ接続します。これにより、長いポゼッションをフェーズ数だけ重複計上する問題を防ぎます。

## 2. 状態空間

```text
s = (origin, location, side, phase_bucket)
```

`phase_bucket`を含める理由は、同じ位置・起点でも1フェーズ目と10フェーズ目では守備配置、疲労、攻撃構造が異なり、位置と起点だけではマルコフ性が弱くなるためです。

## 3. 階層的平滑化

状態ごとの標本数が少ないため、遷移確率を単純な比率で推定しません。

### 吸収確率

```text
p_absorb(s) = (n_absorb(s) + αa πa) / (n_total(s) + αa)
```

### 継続先

```text
P(s' | continue,s)
 = (n(s→s') + αt πt(s')) / (n_continue(s) + αt)
```

### 吸収結果

```text
P(o | absorb,s)
 = (n(s→o) + αo πo(o)) / (n_absorb(s) + αo)
```

事前分布は、可能な場合は同じポゼッション起点・フェーズ帯の観測分布へ縮約し、標本がない場合のみ全体分布へ戻します。

## 4. マルコフ報酬過程

```text
Q[s,s'] = (1-p_absorb(s)) P(s'|continue,s)
B[s,o]  = p_absorb(s) P(o|absorb,s)
```

吸収確率行列は、

```text
H = (I-Q)^(-1) B
```

EPVは、

```text
V = H u
```

です。

## 5. 不確実性

独立なフェーズとしてブートストラップすると、同一試合内の相関を無視します。そのため、試合を単位として復元抽出するクラスターブートストラップを使います。

## 6. 検証

- 遷移確率の各行が1になること
- EPVのBellman残差
- 行列のスペクトル半径が1未満であること
- 試合単位交差検証による終端得点予測
- ブートストラップ区間の幅とサンプル数の関係

## 7. 解釈

このEPVは因果効果ではありません。チーム力、相手、天候、選手、時間、点差などの交絡が残ります。選択肢比較には、アクションデータと方策・結果モデルを追加したMDPまたはオフポリシー評価が必要です。
